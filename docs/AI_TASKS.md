## 차트 인터랙션 (Phase 2) 🔲 보류

> 차트 클릭 시 가로 전체화면 + 드래그 인터랙션. 추후 작업 예정.

### 배경
- 현재 차트는 Canvas 기반 정적 렌더링 (터치 인터랙션 없음)
- X축 스트레칭 방식 적용 완료 (각 곡선이 자기 포인트 수 기준으로 전체 폭 채움)

### 구현 계획

#### 1. 가로 전체화면 차트 🔲
- [ ] 차트 영역 클릭 시 가로(Landscape) 전체화면 전환
- [ ] Activity 또는 Dialog + `requestedOrientation = SCREEN_ORIENTATION_LANDSCAPE`
- [ ] 동일 데이터(`CumulativeChartData`)를 확대 렌더링

#### 2. 드래그 수직선 + Y값 표시 🔲
- [ ] 터치/드래그 시 수직선이 X 위치를 따라감
- [ ] 수직선 위치의 각 곡선 Y값을 툴팁으로 표시 (예: "₩352,000")
- [ ] 스트레칭된 X 좌표에서 선형 보간으로 Y값 산출:
  - 터치 X → 0~1 비율 변환
  - 비율 × (포인트 수 - 1) = 소수점 인덱스
  - 소수점 인덱스에서 선형 보간 → 정확한 Y값

#### 3. 차트 라이브러리 검토 🔲
- [ ] **Vico** (Compose-native, 가장 활발) — 드래그 마커, 애니메이션, 커스텀 축 지원
- [ ] **YCharts** (Compose-native, 심플)
- [ ] 라이브러리 도입 시 기존 Canvas `CumulativeChartCompose` 대체 필요
- [ ] 직접 구현도 가능하나 코드량 증가 (터치 핸들링+보간+애니메이션)

### 핵심 설계 결정
- X축 스트레칭과 호환: 그래프 좌표 기반으로 Y값을 읽으므로 날짜 불일치 무관
- 표시 값은 "추상 그래프 Y값" (실제 날짜가 아닌 곡선 위의 금액)
- 복잡도: 라이브러리 사용 시 중간, 직접 구현 시 중상
